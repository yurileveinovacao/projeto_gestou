### Contexto

Após as Fases 1-4 entregues nos dias anteriores, esta sessão concluiu o MVP do módulo RPA com as 2 fases finais (5 e 6) e fez teste E2E pela UI via Playwright pra validar o fluxo completo.

### Decisão arquitetural

- **Fase 5 — Aceite digital "Opção B"**: tela pública `/app/rpa_aceite.php` SEM login. Autenticação dupla via token (URL) + CPF (input). Termo de uso obrigatório, confirmação dupla via modal SweetAlert, captura de IP + user-agent + timestamp, geração de PDF de evidência com declaração formal referenciando Lei 14.063/2020 art. 4º I.
- **Token de aceite**: SHA-256 (64 chars) gerado por `random_bytes(32)`, validade 7 dias, invalidado após uso único. Coluna nova em GESRPA (`token_aceite`, `token_validade`, `token_invalidado_em`) via migration `alter_gesrpa_add_token_aceite.sql`.
- **Disparo automático**: ao aprovar RPA (Fase 4), `rpa_aprovar_post.php` chama `enviarEmailAceiteAutonomo` (não-blocking). Token é gerado mesmo se SMTP falhar — admin pode pegar pelo banco em local ou reenviar via botão.
- **Botão "Reenviar email de aceite"**: visível em `rpa_alterar.php` quando status=autorizado. Invalida token anterior, gera novo, reenvia.
- **Fase 6 — Status financeiro**: dois novos botões em `rpa_alterar.php`:
  - "Enviar para financeiro" (amarelo, status assinado → enviado_fin, grava data_envio_fin = hoje)
  - "Marcar como pago" (verde, status enviado_fin → pago, modal com input date pra data_pgto)
- **Export Excel mensal**: controller `rpa_exportar_excel_get.php` via PhpSpreadsheet (já no projeto). 18 colunas no formato exato da planilha `RPA.MARÇO.xlsx` da Jéssica + resumo no rodapé (total RPAs, bruto, líquido). Filename `RPAs_{MES_NOME}_{ANO}.xlsx`.
- **Aba RPA em dados_cadastrais**: nova tab com valor padrão, INSS % (readonly fixo 12,36), limites alerta/bloqueio, editor HTML dos 3 templates (autorização, contrato, recibo). Vazio = usa default embutido no helper. Lista de placeholders disponíveis no topo. AJAX submit pra novo controller `dados_cadastrais_rpa_post.php`.
- **Cards no dashboard**: 2 novos cards em `admin/index.php` (carousel slick) — "RPAs pendentes de aprovação" (amarelo) e "RPAs aguardando pagamento" (info). Renderização condicional `> 0`. Links pra `rpas.php?status=...` filtrado. Nova função DAL `selectGESRPA_count_por_status`.
