# Gestou — Progress Log (Fase 4B+)
Início: 2026-03-05

## Codebase Patterns
- Sistema PHP 7.4 puro, sem framework — todos os requires são manuais
- Duas conexões ao banco em paralelo: $conn (pg_connect nativa) e $pdo (PDO)
- Cada módulo (admin/, app/, master/) tem seu próprio conexao.php e conexao_pdo.php
- iuds_pdo.php é o Data Access Layer: admin (10.750 linhas), app (2.404), master (6.532)
- restrito.php é o guard de sessão em cada módulo — tem session_start()
- PHPMailer centralizado em config/mail.php com configureMailer(&$mail) — lê SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM, SMTP_FROM_NAME via getenv()
- URLs centralizadas em config/app.php: $app_url (getenv APP_URL), $contact_email (getenv CONTACT_EMAIL)
- Redirects usam caminhos relativos (/admin/login); emails/QR/print usam $app_url
- Conexão files legados (conexao.php, conexao_pdo.php) redirecionam para config/database.php
- OCR: config/ocr.php com googleVisionOCR() — envia PDF base64 para Google Vision files:annotate, converte para formato Azure
- 83 templates em admin/layout/ (holerite/irrf/ponto) esperam formato Azure: analyzeResult.readResults[n].page, .lines[m].text
- Google Vision files:annotate suporta até 5 páginas por request — config/ocr.php faz batching automático
- Vendors no .gitignore — não estão no repo, serão copiados no Docker build
- Vendors por módulo: admin/ (~341MB, 10 dirs), app/ (~87MB, 6 dirs), master/ (~50MB, 3 dirs), raiz (~30MB, vendor/ + vendor_envio_email/)
- Docker: php:7.4-apache, porta 8080 (Cloud Run), mod_rewrite + AllowOverride All para .htaccess
- Uploads: upload/ (8.7GB), admin/uploads/ (36MB), master/upload/ — montados via GCS FUSE
- Deploy: Cloud Run gen2, us-central1, Unix socket para Cloud SQL
