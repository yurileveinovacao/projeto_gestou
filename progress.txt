# Ralph Progress Log — Gestou Migration
Started: 2026-02-26

## Codebase Patterns
- Sistema PHP 7.4 puro, sem framework — todos os requires são manuais
- Duas conexões ao banco em paralelo: $conn (pg_connect nativa) e $pdo (PDO)
- Cada módulo (admin/, app/, master/) tem seu próprio conexao.php e conexao_pdo.php
- iuds_pdo.php é o Data Access Layer: admin (10.750 linhas), app (2.404), master (6.532)
- restrito.php é o guard de sessão em cada módulo — tem session_start()
- PHPMailer centralizado em config/mail.php com configureMailer(&$mail) — lê SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM, SMTP_FROM_NAME via getenv()
- URLs centralizadas em config/app.php: $app_url (getenv APP_URL), $contact_email (getenv CONTACT_EMAIL)
- Redirects usam caminhos relativos (/admin/login); emails/QR/print usam $app_url
- Conexão files legados (conexao.php, conexao_pdo.php) redirecionam para config/database.php
- OCR: admin/vision_computer.php e admin/controller/vision_computer_irrf_post.php chamam Azure Vision
- 83 templates em admin/layout/ (holerite/irrf/ponto) esperam formato Azure readResults
- Vendors no .gitignore — não estão no repo, serão copiados no Docker build
- Arquivos .example na raiz: conexao.php.example e conexao_pdo.php.example mostram formato das conexões

## 2026-02-26 - MIG-006 - Implementar sessões via PostgreSQL
- Criado config/schema/sessions.sql com CREATE TABLE php_sessions (id VARCHAR(128) PK, data TEXT, last_access TIMESTAMP) e índice para GC
- Criado config/session.php com classe PgSessionHandler implementando SessionHandlerInterface
  - Usa $pdo de config/database.php (require_once)
  - Métodos: open, close, read, write (UPSERT via ON CONFLICT), destroy, gc
  - GC usa intervalo PostgreSQL (CAST(:lifetime AS INTERVAL))
  - Registra handler via session_set_save_handler() automaticamente ao ser incluído
- Integrado nos 3 restrito.php (admin, app, master): require_once ANTES do session_start() existente
- Arquivos criados: config/session.php, config/schema/sessions.sql
- Arquivos alterados: admin/restrito.php, app/restrito.php, master/restrito.php

  - config/schema/sessions.sql existe com CREATE TABLE php_sessions: PASS
  - admin/restrito.php require session.php ANTES de session_start(): PASS

  - master/restrito.php require session.php ANTES de session_start(): PASS
  - grep PgSessionHandler config/session.php retorna resultado: PASS
- **Learnings:**
  - PostgreSQL UPSERT: `INSERT ... ON CONFLICT (id) DO UPDATE SET ...` — ideal para write de sessão
  - GC com intervalo: `NOW() - CAST('1440 seconds' AS INTERVAL)` — mais legível que timestamp math
  - session_set_save_handler($handler, true) — segundo param true registra session_write_close como shutdown function
  - require_once garante que múltiplos requires de restrito.php (que inclui session.php → database.php) não criam conexões duplicadas
